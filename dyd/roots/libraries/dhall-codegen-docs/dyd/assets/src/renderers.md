# Renderers

## About

Renderers are Dhall functions that take a `Document.Type` and return generated source text. Use `dhall text` to evaluate a renderer and write the generated output to a file.

Given a document file such as `my-document.dhall`:

```dhall
let Grammar = ./dhall-codegen/grammar.dhall

let Document = Grammar.Document

let Person = ./schemas/person.dhall

let Organization = ./schemas/organization.dhall

in  Document::{
    , headers =
      [ "// This file was generated by dhall-codegen."
      , "// Do not edit this file directly."
      ]
    , schemas = [ Person, Organization ]
    }
```

You can render it inline from the CLI:

```bash
dhall text \
  --expression 'let r = ./dhall-codegen/renderer-typescript/package.dhall
               in  r.render r.options::{=} (./my-document.dhall)'
```

Or create a reusable renderer file such as `ts-output.dhall`:

```dhall
let TypeScript = ./dhall-codegen/renderer-typescript/package.dhall

let document = ./my-document.dhall

in  TypeScript.render TypeScript.options::{=} document
```

Then evaluate it with:

```bash
dhall text --file ./ts-output.dhall --output ./generated.ts
```

Most renderers run the [lift transformer](./transformers.md) before rendering. This turns nested records into named root definitions so targets can emit stable type names instead of deeply nested inline shapes. The JSON Schema renderer also runs a JSON-Schema-specific lowering transform before lifting.

## Common Conventions

- Root schema names come from `root.meta.name` and become generated type or binding names.
- `Document.headers` are emitted into generated source where the target supports plain text headers.
- Field names are preserved for serialized formats such as JSON Schema and JSON tags.
- Descriptions become target-specific documentation when supported: JSDoc, JSON Schema `description`, Zod `.describe(...)`, Go comments, or Python comments.
- Recursive schemas require a renderer mode that can represent recursion. For Zod, use one of the `ZOD_*_RECURSION` variants.

## TypeScript

**Path:** `dhall-codegen/renderer-typescript/package.dhall`

### API

```dhall
let TS = ./dhall-codegen/renderer-typescript/package.dhall

in  TS.render TS.options::{=} myDocument
```

### Options

```dhall
TS.options.Type =
  { indent : Text
  , break : Text
  , prefix : Text
  }
```

- `indent` and `break` control formatting.
- `prefix` is prepended to exported root type names.

### Output Shape

Each root renders as an exported type alias:

```ts
export type Person =
	{
		name : string;
	};
```

Descriptions are emitted as JSDoc comments.

## TypeScript + Zod

**Path:** `dhall-codegen/renderer-typescript-zod/package.dhall`

### API

```dhall
let Z = ./dhall-codegen/renderer-typescript-zod/package.dhall

in  Z.render Z.options::{=} myDocument
```

Use a recursion variant for recursive schemas:

```dhall
Z.render
  Z.options::{ variant = Z.variant.ZOD_3_RECURSION }
  myDocument
```

### Options

```dhall
Z.options.Type =
  { indent : Text
  , break : Text
  , variant : Z.variant
  }

Z.variant = < ZOD_3 | ZOD_3_RECURSION | ZOD_4 | ZOD_4_RECURSION >
```

- `ZOD_3` and `ZOD_4` choose the target Zod API style.
- `ZOD_*_RECURSION` emits `z.lazy(() => ...)` wrappers and standalone TypeScript types prefixed with `T` to break recursive typing cycles.
- Descriptions are attached with `.describe(...)`.

### Output Shape

Non-recursive roots render as Zod values plus inferred types. Recursive roots render as explicit `z.ZodType<T...>` values using `z.lazy`.

```ts
export const Person = z.object({
	name : z.string(),
});
export type TPerson = z.infer<typeof Person>
```

## JSON Schema

**Path:** `dhall-codegen/renderer-json-schema/package.dhall`

### API

```dhall
let JS = ./dhall-codegen/renderer-json-schema/package.dhall

in  JS.render JS.options::{=} myDocument
```

### Options

```dhall
JS.options.Type =
  { index : Natural
  , depth : Natural
  , indent : Text
  , break : Text
  , variant : JS.variant
  , main : Optional Text
  }

JS.variant = < DRAFT_2020_12 | DRAFT_07 >
```

- `variant` chooses the JSON Schema draft.
- `main = Some "RootName"` sets the top-level `$ref` to a specific root definition.
- `indent` and `break` control formatting.
- `index` and `depth` are renderer context fields; most callers should use their defaults.

### Transform Pipeline

JSON Schema applies an additional transformer before lifting:

1. `transformer-json-schema` lowers constructs JSON Schema cannot express directly.
2. `transformer-lift` lifts nested records to top-level definitions.

See [Transformers](./transformers.md) for details.

### Output Shape

- Draft 2020-12 uses `$defs`; draft-07 uses `definitions`.
- `time` renders as a string with `format: "date-time"`.
- Text variants such as email and URL render as string formats.
- Descriptions render as JSON Schema `description` fields.

The renderer includes a placeholder `$id`. Treat it as an example and replace it as needed.

## Python (Pydantic, Python 3.8+)

**Path:** `dhall-codegen/renderer-python-3.8/package.dhall`

### API

```dhall
let Py = ./dhall-codegen/renderer-python-3.8/package.dhall

in  Py.render myDocument
```

### Options

The Python renderer does not currently expose a public options record. It renders with four-space indentation and newline breaks.

### Output Shape

Records render as Pydantic models:

```python
class Person(BaseModel):
    name : str
```

Notes:

- Output includes imports for Pydantic, common typing helpers, and Python 3.8 compatibility.
- Output includes a small runtime helper to emulate `allOf` intersection checks.
- Descriptions render as Python comments.

## Go

**Path:** `dhall-codegen/renderer-golang/package.dhall`

### API

```dhall
let Go = ./dhall-codegen/renderer-golang/package.dhall

in  Go.render Go.options::{ package = Some "models" } myDocument
```

### Options

```dhall
Go.options.Type =
  { prefix : Text
  , package : Optional Text
  , emitDescriptions : Bool
  }
```

- `prefix` is prepended to exported type names.
- `package = Some "models"` emits a Go `package models` line.
- `emitDescriptions` controls whether schema descriptions are emitted as Go comments.

### Output Shape

Records render as Go structs with JSON tags:

```go
package models

import "time"

type Person struct {
	DateOfBirth time.Time `json:"date_of_birth"`
	Friends []Person `json:"friends"`
	Name string `json:"name"`
	ContactEmail *string `json:"contact_email,omitempty"`
}
```

Field names are exported and converted to Go-style names. JSON tags preserve the schema field names. The renderer imports `time` when schemas use time values.

## Dhall

**Path:** `dhall-codegen/renderer-dhall/package.dhall`

### API

```dhall
let D = ./dhall-codegen/renderer-dhall/package.dhall

in  D.render myDocument
```

Also available:

```dhall
D.renderCompact myDocument
```

### Options

The Dhall renderer does not currently expose a public options record. Use `render` for normal formatting or `renderCompact` for compact formatting.

### Output Shape

The Dhall renderer emits Dhall type definitions for each root schema.

```dhall
let Person =
      { name : Text }

in  { Person }
```

Notes:

- Root schema names become generated Dhall binding names.
- Record schemas render as Dhall record types.
- References render as references to other generated Dhall bindings.
- `renderCompact` emits the same definitions with compact formatting.
